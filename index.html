Most dangerous failure — and how I caught it最危险的失败 —— 以及我怎么抓到它

The silent LEFT JOIN → INNER JOIN collapse静默的 LEFT JOIN → INNER JOIN 塌缩

An early SQL referenced a left-joined column in WHERE. When the key was NULL, NULL IN (...) dropped the row — turning the LEFT JOIN into an INNER JOIN, shrinking the denominator and inflating conversion rate. A syntax error screams; a wrong denominator looks perfectly normal and quietly poisons the decision. I made the join invariant explicit so it can't regress.早期一条 SQL 在 WHERE 里引用了被 left-join 的列。当键为 NULL 时，NULL IN (...) 会丢弃该行 — 于是 LEFT JOIN 退化成 INNER JOIN，分母变小、转化率被抬高。语法错误会报警；分母错误看起来完全正常，却悄悄污染决策。我把这个 join 不变量显式化，让它无法回退。

AB-Test Text-to-SQL WorkflowAB 测试 Text-to-SQL 工作流

Multi-region traffic & funnel analysis pipeline多区域流量与漏斗分析流水线

A deterministic low-code workflow for AB-test analysis across 8 regions. The key engineering judgment: shrink the LLM's job to Stage 1 (variable extraction) and make everything downstream deterministic — flexible enough to take natural language, reliable enough for production.一个覆盖 8 个区域的确定性低代码工作流。关键的工程判断：把 LLM 的职责收缩到 Stage 1（变量抽取），下游全部确定性执行——既能接收自然语言，又足够可靠可上生产。

Query latency: ~300 s → ~2 s查询延迟：~300 秒 → ~2 秒

Raw-table scan ~300 s (timeouts)
Pre-agg mart ~2 s

Earlier filtering, fewer redundant joins, and migrating expensive raw-table queries onto pre-aggregated data marts.更早过滤、减少无效 join，并把昂贵的原始表查询迁移到预聚合数据集市。

Honest framing: this speedup is data modeling & query optimization (pre-agg marts, predicate pushdown, join control) — not the LLM. The agent automates the analysis chain; it does not accelerate the database.诚实地说：这个提速来自数据建模与查询优化（预聚合集市、谓词下推、join 控制）——不是 LLM。Agent 自动化的是分析链路，并不加速数据库本身。

Research Sketch

Early-stage proposal for discussion

When students use AI tools for learning, how does the format of AI-generated feedback — direct explanation vs. scaffolded self-evaluation prompt — affect conceptual retention and metacognitive awareness?

Motivation

Teaching 300+ students, I saw fluent AI explanations help complete tasks while bypassing the cognitive effort needed for durable understanding. If the form and timing of intervention shapes receptivity, can interface design make learners do the cognitive work required to actually learn?

Three Interface Conditions

1Direct Explanation
Complete answer delivered immediately.
2Socratic Scaffolding
Guided questions before revealing answer.
3Self-Evaluation First
Learner predicts & rates confidence first.

Measures

Conceptual RetentionImmediate & delayed post-tests
Metacognitive AccuracyPredicted vs. actual performance
Learning ProcessInteraction logs, time-on-task, help-seeking
User ExperienceEffort, usefulness, frustration, trust

Expected Contribution

An interface design pattern for LLM-based learning tools that balances answer quality with learner reflection, self-evaluation, and durable understanding.
